01

Bibliotecas e definições

As duas primeiras linhas trazem as ferramentas prontas: uma para conversar com o sensor DHT22, outra para controlar o servo motor — essa já vem instalada por padrão no Arduino.

#include <DHT.h>
#include <Servo.h>

#define PINO_DHT 2
#define TIPO_DHT DHT22
#define PINO_SERVO 9
#define UMIDADE_LIMITE 70.0
Por que #define? Cria apelidos fixos pros números. Se um dia você trocar o sensor de pino, muda em um lugar só — o resto do código continua chamando PINO_DHT.
02

Objetos e estado

Aqui nascem os dois "personagens" do código: o sensor e o servo. E uma variável que guarda se a janela está fechada ou não — essa é a memória do sistema.

DHT dht(PINO_DHT, TIPO_DHT);
Servo servoJanela;

bool janelaFechada = false;

const int ANGULO_ABERTA = 0;
const int ANGULO_FECHADA = 90;
O papel do bool janelaFechada: sem essa variável, o Arduino não saberia se já mandou fechar a janela — e ficaria repetindo o comando toda hora, mesmo sem necessidade.
03

setup() — roda uma vez

Liga a comunicação serial, inicializa o sensor, conecta o servo ao pino físico e já deixa a janela na posição aberta assim que o Arduino é ligado.

void setup() {
  Serial.begin(9600);
  dht.begin();
  servoJanela.attach(PINO_SERVO);
  servoJanela.write(ANGULO_ABERTA);
}
04

Lendo o sensor, com segurança

A cada volta do loop(), o Arduino pergunta ao DHT22 qual a umidade e temperatura atuais. Mas sensores falham às vezes — por isso existe a checagem isnan() (is not a number).

float umidade = dht.readHumidity();
float temperatura = dht.readTemperature();

if (isnan(umidade) || isnan(temperatura)) {
  Serial.println("Erro ao ler o sensor!");
  delay(2000);
  return;
}
O return aqui é o herói silencioso: se a leitura vier corrompida, o código pula todo o resto do loop e tenta de novo na próxima volta, em vez de agir com base num dado errado.
05

A decisão: fechar ou abrir

Este é o cérebro do projeto. Duas condições, uma oposta à outra, cada uma só age quando o estado muda — é aqui que a variável janelaFechada prova seu valor.

if (umidade >= UMIDADE_LIMITE && !janelaFechada) {
  servoJanela.write(ANGULO_FECHADA);
  janelaFechada = true;
}
else if (umidade < UMIDADE_LIMITE && janelaFechada) {
  servoJanela.write(ANGULO_ABERTA);
  janelaFechada = false;
}
Por que o !janelaFechada importa: sem ele, o servo receberia o comando "feche" a cada 2 segundos enquanto a umidade estivesse alta — mesmo já fechado. Isso causa tremulação no motor e desgaste do mecanismo com o tempo.
06

O ritmo do loop

Por fim, uma pausa de 2 segundos antes de repetir tudo. O DHT22 não é um sensor rápido — ler com pressa demais só traz mais erros.

delay(2000);
